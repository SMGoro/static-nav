🔧 Jinja模板功能测试指南

本页面提供了测试新增Jinja模板功能的详细说明和示例。

1. Jinja模板功能概述
Jinja模板功能是对URL自动获取功能的增强，使用jinja模板语法来智能解析网站信息。

主要功能特性：

  • Jinja模板解析：使用jinja模板语法分析网站内容
  • 技术栈检测：自动识别网站使用的技术栈
  • 社交媒体提取：提取Twitter、Facebook、LinkedIn等社交媒体信息
  • 联系信息识别：自动识别邮箱、电话、地址等联系信息
  • 智能分类：基于网站内容智能分类
  • 标签推荐：根据网站特性推荐相关标签
2. Jinja模板语法示例
以下是Jinja模板的核心语法示例，用于智能解析不同类型的网站。
{% if 'github' in url %} 网站类型：开发工具 分类：开发工具 标签：['Git', '代码托管', '开源', '协作'] 特性：['版本控制', '代码审查', '项目管理', 'CI/CD'] {% elif 'figma' in url %} 网站类型：设计工具 分类：设计资源 标签：['设计', 'UI/UX', '原型设计', '协作'] 特性：['实时协作', '原型设计', '设计系统', '插件生态'] {% elif 'notion' in url %} 网站类型：效率工具 分类：效率工具 标签：['笔记', '项目管理', '知识管理', '协作'] 特性：['块级编辑', '数据库', '模板库', '多平台同步'] {% else %} 网站类型：{{ category | default('其他') }} 分类：{{ category | default('其他') }} 标签：{{ tags | default(['网站']) }} 特性：{{ features | default([]) }} {% endif %}
3. 技术栈检测功能
自动检测网站使用的技术栈，支持多种主流技术。

支持的技术栈：

  • 前端框架：React, Vue.js, Angular
  • 后端语言：Node.js, Python, Ruby, PHP
  • 框架：Django, Flask, Rails, WordPress
  • 云服务：AWS, Google Cloud, Azure
  • 容器化：Docker, Kubernetes
  • 数据库：MySQL, PostgreSQL, MongoDB, Redis
  • 服务器：Nginx, Apache
4. 社交媒体信息提取
自动提取网站的社交媒体信息。

支持的社交媒体平台：

  • Twitter：提取twitter:site元数据
  • Facebook：提取fb:app_id元数据
  • LinkedIn：提取linkedin:company元数据
  • Instagram：提取instagram:site元数据
5. 联系信息识别
智能识别网站中的联系信息。

识别类型：

  • 邮箱地址：使用正则表达式匹配邮箱格式
  • 电话号码：识别各种格式的电话号码
  • 地址信息：从HTML中提取地址相关元素

7. 测试步骤
按照以下步骤测试Jinja模板功能。
  1. 进入网站添加页面
  2. 选择"URL自动获取"标签页
  3. 输入上述测试URL中的一个
  4. 点击"自动获取"按钮
  5. 等待jinja模板解析完成
  6. 查看自动填充的网站信息
  7. 检查浏览器控制台的详细解析结果
  8. 验证技术栈、社交媒体、联系信息等是否正确提取
8. 预期结果
测试成功后应该看到以下结果。

成功指标：

  • 网站标题和描述正确提取
  • 分类和标签符合网站类型
  • 技术栈正确识别（如果适用）
  • 社交媒体信息正确提取
  • 联系信息正确识别
  • 控制台显示详细的解析日志
  • 网站图标正确匹配
9. 调试信息
在浏览器控制台中查看详细的解析信息。

控制台输出示例：

检测到的技术栈: ['React', 'Node.js', 'MongoDB']
社交媒体信息: {twitter: '@example', facebook: '123456789'}
联系信息: {email: 'contact@example.com', phone: '+1-234-567-8900'}

注意事项：

  • 确保网络连接稳定
  • 某些网站可能有访问限制
  • 技术栈检测基于HTML内容分析
  • 社交媒体信息需要网站提供相应的元数据
  • 联系信息识别可能不是100%准确
  • Jinja模板解析结果可能因网站结构而异
10. 故障排除
如果遇到问题，请检查以下项目。

常见问题：

  • 无法获取网站内容：检查网络连接和URL是否正确
  • 解析结果不准确：某些网站结构可能不支持自动解析
  • 技术栈未检测到：网站可能使用了自定义技术或隐藏了技术信息
  • 社交媒体信息缺失：网站可能没有提供相应的元数据
  • 联系信息识别失败：信息可能以非标准格式存在
